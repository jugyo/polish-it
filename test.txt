# システム仕様書みたいなやつ

## はじめに

このドキュメントは、ユーザー管理システムの仕様をまとめたものです。ぜひご覧ください。

## システムの概要とか

このシステムは、ユーザーの情報を管理するためのものになります。具体的には、ユーザーの登録とか、削除とか、あと編集とかができます。データベースを使っていて、MySQLを使う予定だったけど、PostgreSQLになるかもしれないです。まだ決まってないです。

## 機能について

### ユーザー登録機能

ユーザーを登録できます。名前とメールアドレスと、パスワードが必要です。パスワードは8文字以上にしてほしいです。たぶん。あと、メールアドレスは重複しちゃダメです。登録したらメールが飛ぶかもしれないけど、それは後で考えます。

### ユーザー削除機能

ユーザーを消せます。消したら元に戻せないので注意です。でも論理削除にするかもしれないです。物理削除は怖いので。削除するときは確認ダイアログを出したいと思ってます。

### ユーザー編集機能

ユーザーの情報を変えれます。名前とメールアドレスは変えれます。パスワードも変えれるけど、古いパスワードを入れないとダメにしたいです。

### ログイン機能

ログインができます。メールアドレスとパスワードでログインします。間違えたら3回までは許容して、それ以上は30分ロックします。たぶんそうします。セッションは2時間で切れます。いや、1時間かも。

## 画面について

画面は5個くらいあります。ログイン画面と、ユーザー一覧画面と、ユーザー詳細画面と、登録画面と、編集画面です。デザインはまだ決まってなくて、とりあえずBootstrapを使おうと思ってます。レスポンシブにしたいけど、時間があれば。

## API

RESTfulなAPIを作ります。JSONで返します。エンドポイントは以下のような感じです。

- GET /users - ユーザー一覧
- GET /users/:id - ユーザー詳細
- POST /users - ユーザー作成
- PUT /users/:id - ユーザー更新
- DELETE /users/:id - ユーザー削除

認証はJWTを使うと思います。Bearerトークンをヘッダーにつけてください。

## その他

エラーハンドリングはちゃんとやりたいです。500エラーとか出ないようにしたいです。ログも取りたいです。何かあったときに見れるように。

テストは書きたいけど、時間があれば書きます。単体テストとか、E2Eテストとか。カバレッジは80%くらいを目指したいです。
